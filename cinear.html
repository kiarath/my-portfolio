Cine.AR – Streaming Service

A complete web redesign for Cine.AR, an Argentinian-focused free streaming service, enhancing user experience and showcasing local cinema.

RoleUX/UI Designer
Timeline2023
ToolsFigma
Web Entertainment Figma

What is CINE.AR PLAY?

Cine.AR Play is a free streaming platform focused on content produced in Argentina, including films, series, and documentaries. Originally presented as the "Netflix Criollo" (Creole Netflix), it currently features thousands of hours of content and a very varied curation.

Why a Redesign?

Despite its great quantity of content, we found that users have difficulty navigating the platform and finding what they are looking for. The last redesign of the platform was in 2017, and many things have happened since then. We decided to conduct research to see which aspects could be improved.

Context

Currently, a major interest in Argentine Cinema is emerging due to easier access to films, coupled with an increase in interest from recent international awards they have received.

The Problem

Due to market saturation and fierce competition, the Cine.AR Play web application has been left behind compared to innovations introduced by leading platforms like Netflix or Prime Video. This situation makes it difficult for the page to stand out and attract new users.

  • • Redesign the homepage to improve content hierarchy and speed up movie selection.
  • • Add more options to the player to make it more accessible and improve consistency.
02. Requirements (Scope)

Needs Matrix

OpportunityPain PointNeedSolution
Speed up the login process. The user has to manually enter their login details every time. The user wants to enter their account more quickly. Add login options (Google, Facebook).
Redesign the homepage to improve content hierarchy. The homepage is disorganized and difficult to navigate. The user wants to find content that suits their tastes. Add "Recommendations" at the beginning and new content buttons.
Add more options to the player. The current player lacks basic functions like subtitles, playback speed, etc. The user wants a video player with standard streaming functions. Add buttons for subtitles, playback speed, etc.

Feasibility, Viability & Desirability Matrix

Priority 1 (High):

  • • Reduce the number of elements on the homepage.
  • • Add the ability to view the synopsis from the homepage.
  • • Filter series/movies by genre.
  • • Speed up login methods (Google/Facebook).

Priority 2:

  • • Add a "Recommendations" section based on user behavior.
  • • Change the name of "My List" to "My Favorites."

Priority 3:

  • • Add icons to profiles for different users.
  • • Add clear differences between rented and free content.

Priority 4 (Low):

  • • Create a "Trends" section.

03. Structure

Methodology

1st Tree Testing → Cardsorting → 2nd Tree Testing

1st Tree Testing (Original Structure)

Results: Very Low success rate (35%).

Task 1: "You want to see a movie you purchased previously, where would you do it?"

Result: 60% FAILED.

Conclusion: Users were confused by ambiguous labels like "Mi Sala" (My Room).

Task 2: "You're in the mood for a comedy movie, where would you look?"

Result: 80% FAILED.

Conclusion: The path to browse by genre was unclear and fragmented.

Cardsorting

Based on the 1st test, a card sorting exercise was conducted, which resulted in a new, simplified sitemap:

Movies
Series
My List
My Purchases
My Profile
Help

2nd Tree Testing (New Structure)

Results: Very High success rate (95%).

Task 1: "You want to see a movie you purchased previously…"

Result: 90% SUCCESS.

(Path: Home → My Purchases)

Task 2: "You're in the mood for a comedy movie…"

Result: 100% SUCCESS.

(Path: Home → Movies → Comedy)

04. Skeleton

Moderated Testing

Hypothetical Scenario: You come back from work and feel like exploring Argentine cinema, so you decide to test this "Cine.Ar" page.

Task 1: CHOOSE A MOVIE

Objective: The user can choose a movie quickly.

Success Criterion: Reproduce a movie.

Task 2: RENT A MOVIE

Objective: The user can rent a movie.

Success Criterion: Complete the payment process.

Users: 4 users (medium-low tech skills, ages 18-30).

Results: 4/4 (100%) of users successfully completed ALL TASKS.
